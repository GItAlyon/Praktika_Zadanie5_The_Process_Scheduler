# Планировщик процессов и монитор системы

## Описание проекта

Программа представляет собой учебный симулятор планировщика процессов, а также утилиту для просмотра загруженности системы (аналог `top`/`htop`).

### Основные возможности:

1. **Симуляция алгоритмов планирования:**
   - Round Robin (циклический, вытесняющий)
   - Приоритетное планирование с очередями (невытесняющий)
   - SJF (Shortest Job First, невытесняющий)

2. **Визуализация:**
   - Диаграмма Ганта в терминале
   - Таблицы со статистикой процессов
   - График загрузки CPU

3. **Мониторинг системы:**
   - Просмотр реальных процессов Windows
   - Отображение загрузки CPU в реальном времени
   - Обновление данных каждую секунду

4. **Интерактивный режим:**
   - Пошаговый показ работы алгоритмов
   - Сравнение всех алгоритмов на одних данных

## Алгоритмы планирования

### 1. Round Robin (циклический)
- **Тип:** Вытесняющий
- **Принцип работы:** Каждый процесс получает квант времени (по умолчанию 2 единицы). По истечении кванта процесс переходит в конец очереди готовности.
- **Применение:** Интерактивные системы, где важна справедливость распределения CPU.

### 2. Приоритетное планирование с очередями
- **Тип:** Невытесняющий
- **Принцип работы:** Процессы распределяются по трем очередям в зависимости от приоритета (1 - высокий, 2 - средний, 3 - низкий). Процесс выполняется до завершения.
- **Применение:** Системы с разными приоритетами задач (например, системные процессы важнее пользовательских).

### 3. SJF (Shortest Job First)
- **Тип:** Невытесняющий
- **Принцип работы:** Выполняется процесс с наименьшим временем выполнения.
- **Применение:** Пакетная обработка, где нужно минимизировать среднее время ожидания.

## Возможные улучшения

1. **Добавить другие алгоритмы:**
   - SRTF (Shortest Remaining Time First) - вытесняющий SJF
   - Многоуровневая очередь с обратной связью
   - Гарантированное планирование
   - Справедливое планирование

2. **Расширить мониторинг:**
   - Отображение использования памяти
   - Сортировка процессов по загрузке CPU
   - Информация о потоках

3. **Кроссплатформенность:**
   - Добавить поддержку Linux (/proc)
   - Поддержка macOS

4. **Графический интерфейс:**
   - Использовать ncurses для более красивой визуализации
   - Создать GUI-версию

### Примечания
- Программа использует Windows API для мониторинга процессов (<windows.h>, <tlhelp32.h>)
- В Linux доступна только симуляция планировщика
- Для корректного отображения русского языка используется кодировка Windows-1251
